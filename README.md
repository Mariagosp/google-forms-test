# Forms Lite ‚Äî Google Forms Clone

A simplified Google Forms clone built as a monorepo. Supports creating forms with multiple question types, filling them out, and viewing submitted responses.

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| **Front-end** | React 19, TypeScript, Redux Toolkit (+ RTK Query), React Router v7, CSS Modules |
| **Back-end** | NestJS 11, GraphQL (code-first via `@nestjs/graphql` + Apollo), in-memory store |
| **Shared types** | GraphQL Codegen generates TypeScript types from `server/src/schema.gql` directly into the client |
| **Monorepo runner** | `concurrently` at the root |

## Getting Started

### Prerequisites

- Node.js ‚â• 20
- npm ‚â• 10

### Install dependencies

```bash
# Install root devDependencies (concurrently)
npm install

# Install server dependencies
npm install --prefix server

# Install client dependencies
npm install --prefix client
```

### Run in development

```bash
npm run dev
```

This starts both services concurrently:

| Service | URL |
|---------|-----|
| GraphQL API | `http://localhost:3000/graphql` |
| React app | `http://localhost:5173` |

### Regenerate GraphQL types (optional)

The generated file (`client/src/services/generatedApi.ts`) is committed to the repo. If you change the server schema or client `.graphql` files, regenerate it with:

```bash
npm run codegen --prefix client
```

> **Note:** The server must be running before running codegen, since codegen reads the live schema.

## GraphQL API

### Queries

```graphql
forms: [Form!]!                          # All forms
form(id: String!): Form                  # Single form by ID
responses(formId: String!): [Response!]  # All responses for a form
```

### Mutations

```graphql
createForm(title: String!, description: String, questions: [QuestionInput]): Form!
submitResponse(formId: String!, answers: [AnswerInput]): Response!
```

### Question types

`TEXT` ¬∑ `MULTIPLE_CHOICE` ¬∑ `CHECKBOX` ¬∑ `DATE`

## Architecture Decisions

### Why NestJS?
NestJS provides a structured, decorator-based approach that maps cleanly onto GraphQL's code-first pattern. The `@ObjectType` / `@InputType` / `@Resolver` decorators make the schema self-documenting and keep types close to their logic ‚Äî no separate SDL files to keep in sync.

### Why RTK Query instead of Apollo Client for data fetching?
The project uses Redux Toolkit for state management, so RTK Query is the natural fit: one dependency, one cache, one devtools pane. Apollo Client is initialized as a peer (required by `@apollo/client` which ships alongside `graphql-request`) but all actual data fetching goes through RTK Query hooks generated by GraphQL Codegen.

### Why GraphQL Codegen?
Codegen reads `server/src/schema.gql` and all `.graphql` documents in the client, then emits fully-typed RTK Query hooks into `generatedApi.ts`. This means the server schema is the single source of truth for all TypeScript types shared between packages ‚Äî no manual type duplication.
